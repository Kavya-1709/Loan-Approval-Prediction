# Loan Approval Prediction

A machine learning pipeline that predicts whether a loan application will be approved, with a Streamlit web app for real-time, explainable predictions.

## Overview

This project takes applicant financial and demographic data and predicts loan approval outcomes. It goes beyond a simple classifier by using SHAP to explain *why* each prediction was made, and ships as an interactive web app rather than just a notebook.

## Dataset

The dataset (`loanapproval.csv`) contains 1,000 applicant records with the following fields:

| Feature | Description |
|---|---|
| `age` | Applicant's age |
| `gender` | Applicant's gender |
| `marital_status` | Married / Single / Divorced |
| `annual_income` | Annual income |
| `loan_amount` | Requested loan amount |
| `credit_score` | Applicant's credit score |
| `num_dependents` | Number of dependents |
| `existing_loans_count` | Number of existing loans |
| `employment_status` | Employed / Self-employed / Unemployed |
| `loan_approved` | Target variable (1 = approved, 0 = rejected) |

## Approach

1. **Exploratory Data Analysis** — examined feature distributions and correlations, and analyzed how credit score, income, and existing loans interact with age, employment status, marital status, and dependents to affect approval.
2. **Optimal Binning (WOE)** — used weight-of-evidence binning on key numerical features (loan amount, income, credit score, dependents, existing loans) to understand their relationship with approval likelihood.
3. **Feature Engineering** — derived three additional risk signals:
   - `loan_income_ratio` — loan amount relative to income
   - `income_per_dependent` — income adjusted for household size
   - `debt_burden` — existing loans weighted by loan-to-income ratio
4. **Modeling** — trained and tuned three classifiers via `GridSearchCV` with 5-fold stratified cross-validation, optimizing for ROC-AUC:
   - Logistic Regression (with feature scaling)
   - Random Forest
   - XGBoost
5. **Evaluation** — compared models on accuracy, precision, recall, ROC-AUC, and confusion matrices on a held-out test set. **XGBoost** was selected as the best-performing model.
6. **Explainability** — used SHAP (`TreeExplainer`) to generate summary, bar, and waterfall plots showing which features drive individual predictions.
7. **Deployment** — packaged the trained model into a Streamlit app for real-time, interactive predictions.

## Web App

The Streamlit app (`app.py`) lets a user enter applicant details (age, income, credit score, loan amount, dependents, existing loans, employment status, marital status, gender) and returns:
- An **Approved / Rejected** decision
- The **approval probability**
- The **top 3 factors** driving that specific prediction, via SHAP

## Tech Stack

Python · Pandas · Scikit-learn · XGBoost · OptBinning · SHAP · Streamlit · Seaborn/Matplotlib

## Project Structure

```
├── project.ipynb        # EDA, binning, feature engineering, modeling, SHAP analysis
├── loanapproval.csv     # Dataset
├── app.py                # Streamlit app for deployment
├── loan_model.pkl         # Saved best model (generated by notebook)
├── features.pkl           # Saved feature list (generated by notebook)
└── README.md
```

## Running Locally

```bash
pip install streamlit pandas joblib shap numpy xgboost scikit-learn optbinning
streamlit run app.py
```

Make sure `loan_model.pkl` and `features.pkl` (generated at the end of `project.ipynb`) are in the same directory as `app.py`.
